# care-agent-boundary-tests

Инструмент для проверки границ поведения AI-агентов в чувствительных доменах:
образование, помощь, сопровождение, работа с детьми, care-среды.

## Задача

Определять, где агент:
- начинает подменять решения пользователя;
- проявляет скрытую директивность;
- формирует зависимость;
- размывает границы роли;
- использует псевдо-эмпатию как средство влияния.

Это не генератор ответов и не чат-бот.
Это тестовый и исследовательский контур.

## Связь с pressure-verdict

Этот репозиторий — прикладной тестовый стенд.

Логика такая:
- `pressure-verdict` умеет выносить explainable verdict по одному тексту или логу;
- `care-agent-boundary-tests` хранит кейсы;
- затем прогоняет кейсы через `pressure-verdict`;
- и позволяет видеть, какие сценарии дают высокий риск.

## Формат MVP

Каждый кейс описывается как JSON-объект:
- `id`
- `domain`
- `scenario`
- `agent_response`
- `expected_min_verdict`

## Что умеет текущий MVP

- читает файл с кейсами;
- прогоняет все кейсы через локальный движок verdict;
- считает статус `PASS` / `FAIL`;
- печатает человекочитаемый отчёт;
- умеет выводить JSON.
